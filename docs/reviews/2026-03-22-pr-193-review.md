## Key Findings

### P1 — Critical (3)

1. **Double-remove in WaitForAsync catch + finally** — `_waiters.Remove(entry)` called in both `catch` and `finally` on timeout path. All 5 code reviewers flagged independently.
2. **WaitForPublished<T> type matching permanently broken** — `RecordingTransport` calls `Type.GetType()` with a short name (framework writes `messageType.Name`), always returns `null`. Published messages recorded as `typeof(TransportMessage)`.
3. **Clear() produces non-descriptive OperationCanceledException** — `TrySetCanceled()` on pending waiters is indistinguishable from external cancellation. Shared fixture pattern breaks diagnostics.

### P2 — Important (7)

4. **OperationCanceledException recorded as Faulted** — Pipeline decorator catches all exceptions including OCE as faults.
5. **No default timeout on WaitFor\* methods** — Every call site specifies `TimeSpan.FromSeconds(5)`.
6. **ConfigureServices not idempotent** — Double call double-wraps decorators.
7. **CTS creation after waiter registration** — Race window where waiter is live but timeout not armed.
8. **Predicate execution inside _waitersLock** — Deadlock risk if predicate re-enters harness.
9. **Missing Argument.\* guards** — `CreateAsync(configure)` and `WaitForAsync(messageType)` lack null checks.
10. **Bare catch {} swallows serializer bugs** — No logging, cryptic timeout replaces real error.

### P3 — Nice-to-Have (5)

11. **Unnecessary Lazy<IMessagePublisher>** — Wraps a dictionary lookup.
12. **Timestamp doc ambiguity** — Observation time vs creation time not clear.
13. **TestConsumer<T> missing Count/IsEmpty** — Also no doc linking to WaitFor mechanism.
14. **No count-based WaitFor\* overloads** — Fan-out scenarios require manual Task.WhenAll.
15. **Snapshot properties allocate on every access** — ToArray() per property access undocumented.

## Overall Assessment

Well-structured package with sound architecture. Decorator pattern over filter, signal-based waits over polling, and fresh ServiceCollection per harness are all the right calls. The P1s are real correctness issues that should be fixed before merge. The package scores 11/14 on agent-native parity.
