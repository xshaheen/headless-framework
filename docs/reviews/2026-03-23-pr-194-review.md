# Code Review — PR #194: feat(messaging): per-group circuit breaker and adaptive retry backpressure

## Summary

A substantial, well-architected addition. The circuit breaker state machine, timer lifecycle, and transport pause/resume are correctly designed. The generation-counter pattern for timer safety, the per-group locking model, and the `ConsumerPauseGate` `TaskCompletionSource` implementation are all sound. Test coverage is strong.

Five correctness issues block merge. Nine important improvements should be addressed. Eight simplifications are nice-to-have.

## Key Findings

### 🔴 P1 — Critical (Blocks Merge)

**074** — `CircuitBreakerMetrics._SafeTag` returns `_unknown` for ALL groups before `RegisterKnownGroups` is called — metrics permanently wrong for pre-startup trips.

**078** — `ReportSuccessAsync` unconditionally resets `ConsecutiveFailures` even in `Open` state — can erase failure history, causing premature circuit closure.

**081** — `ResumeTask` assigned after `Task.Run` — `DisposeAsync` can miss in-flight resume callback; transport resumes after shutdown.

**084** — `RabbitMqConsumerClient.ResumeAsync` uses null-forgiving `_channel!` — `NullReferenceException` after disposal.

**088** — `ResetAsync` signature in `docs/llms/messaging.txt` includes non-existent `CancellationToken` parameter — all agent-generated recovery code fails to compile.
