# Code Review Summary — PR #194 (Pass 2)

Post-fix review after resolving todos 012–018. Six merge-blocking issues remain: dead
`ProbePermit` semaphore left in `GroupCircuitState`, OTel metric calls inside the group
lock, `IValidateOptions<T>` registrations that are never invoked, unbounded OTel cardinality
from attacker-controlled group names, NATS semaphore leak in fire-and-forget callbacks, and
`EscalationLevel` incremented after it is read.

## Reviewers Used

- strict-dotnet-reviewer — correctness, DI wiring, runtime behavior
- pragmatic-dotnet-reviewer — API design, ergonomics, operability
- security-sentinel — failure amplification and abuse resistance
- performance-oracle — backpressure, throughput, resource usage
- code-simplicity-reviewer — dead abstractions and unnecessary complexity
- agent-native-reviewer — agent discoverability of new configuration surface
- learnings-researcher — docs/solutions pattern scan

## Key Findings

### 🔴 P1 — Critical (Blocks Merge)

- **028** — `GroupCircuitState._probePermit` (SemaphoreSlim) and `_TryReleaseProbeSemaphore` are dead code; nothing acquires the semaphore so every release is a no-op.
- **031** — `metrics.RecordTrip/RecordOpenDuration` called inside the `lock(_lock)` critical section in `CircuitBreakerStateManager`; inflates lock hold time under OTel exporters.
- **035** — `IValidateOptions<ConsumerCircuitBreakerOptions>` and `IValidateOptions<RetryProcessorOptions>` registered but never invoked because options are snapshotted via `Options.Create()`, bypassing the options pipeline.
- **040** — `_GetOrAddState(groupName)` called from OTel measurement callbacks with attacker-controlled input; unbounded `ConcurrentDictionary` growth → OTel cardinality explosion and OOM.
- **043** — NATS semaphore acquired before fire-and-forget `Task.Run` but released inside it; an exception in the callback permanently stalls the consumer by leaking the semaphore slot.
- **047** — `EscalationLevel` incremented after `_GetOpenDuration(state.EscalationLevel)` reads it; first escalation always uses level 0 duration and the increment is semantically misleading.

### 🟡 P2 — Important

- **029** — `_locks ConcurrentDictionary<string, object>` in `ConsumerCircuitBreakerRegistry` is redundant; move to `GroupCircuitState.SyncLock` field.
- **032** — `RegisterGroupCallbacks` called after consumer background tasks start; `OnPause` callback is null during any early circuit trip.
- **034** — `_ = client.PauseAsync()` in `GroupHandle.AddClient` silently swallows exceptions; log or await properly.
- **037** — `ProbePermit` SemaphoreSlim and `OpenTimer` in `GroupCircuitState` never disposed; `CircuitBreakerStateManager` does not implement `IDisposable`.
- **039** — `IsOpen` property declared on both `ICircuitBreakerStateManager` and `ICircuitBreakerMonitor`; unnecessary duplication.
- **042** — `Configure<MessagingOptions>` callback in `Setup.cs` never copies `CircuitBreaker` or `RetryProcessor` sub-options, so consumer-level overrides are silently discarded via the DI path.
- **045** — `SuccessfulCyclesToResetEscalation` has no validator; value of 0 silently disables escalation reset.
- **048** — If the resume callback throws during HalfOpen→Closed transition, circuit stays in HalfOpen forever with no re-trip path.
- **049** — No `ResetAsync` on `ICircuitBreakerMonitor`; operators and agents cannot short-circuit the open-duration timer.
- **050** — `_currentInterval`, `_consecutiveHealthyCycles`, `_consecutiveCleanCycles` in `MessageNeedToRetryProcessor` are not thread-safe; concurrent `ProcessAsync` calls race on them.

### 🔵 P3 — Nice-to-Have

- **027** — `_lookbackSeconds` field holds a `TimeSpan`, not seconds; rename to `_lookbackWindow`.
- **030** — `RecordOpenDuration` histogram unit is seconds but parameter is named `durationMs`; misleading.
- **033** — No `GetAllStates()` on `ICircuitBreakerMonitor`; agents cannot enumerate all group states.
- **036** — No OTel observable gauge for current circuit state per group; makes dashboards harder to build.
- **038** — `_semaphore.WaitAsync()` in NATS consumer missing cancellation token.
- **041** — `topics.Select(t => t)` no-op LINQ in `ConsumerRegister.ExecuteAsync`.
- **044** — `OnOpenTimerElapsed` dispose-vs-in-flight-callback invariant is undocumented; add cross-reference comment.
- **046** — Two-counter adaptive polling logic in `_AdjustPollingInterval` has confusing zero-message-branch overlap; simplify.
