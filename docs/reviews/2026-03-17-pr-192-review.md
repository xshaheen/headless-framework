## Key Findings

### P1 Critical (7)
1. `GetOrSetArrayAsync` returns `null` after successful cache write (silent data loss)
2. `new HttpClient()` per request in ping endpoint (socket exhaustion)
3. K8s `_GetPortByName` inverted null check (port 0 for all K8s nodes)
4. Redis node registry race condition (no atomicity in multi-node)
5. `static` fields on generic `JobsInMemoryPersistenceProvider` (test isolation)
6. `startPolling` TOCTOU race (double polling loops)
7. `/negotiate` auth middleware bypass via `Contains` (auth bypass vector)

### P2 Important (8)
1. `access_token` query string leaks to logs
2. GatewayProxy cache key collision + missing CancellationToken
3. Frontend race conditions (concurrent loads, ping deadlock, double-execute)
4. N+1 Redis reads + redundant DB queries
5. Missing WithSummary on all messaging endpoints
6. AuthConfig.CustomValidator Func closure risk + mutable AuthResult
7. SPA helper duplication (~100 LOC)
8. HeartbeatBackgroundService silently dies + event subscription leak

### P3 Nice-to-Have (4)
1. Demo security issues (hardcoded JWT, missing safety comments)
2. Frontend type safety (any types, falsy-zero, regex escape, non-reactive computed)
3. Dead code and public API surface cleanup
4. Style/idiom issues (Tuple types, dictionary merge, rate limiting)
