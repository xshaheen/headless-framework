# Code Review Summary — PR #187: feat: add property-level audit log subsystem

## Overview

Solid architecture overall. The abstraction+provider split, opt-in semantics, atomic EF commit, and sensitive-data pipeline are all well-designed. Two pre-existing P1 bugs (store-generated key capture, multi-DbContext binding) were tracked during development and remain open. This review adds 3 new P1 findings and 15 P2/P3 findings.

**Merge is blocked by 3 new P1 findings** (plus the 2 pre-existing P1 todos tracked in the PR).

## Key Findings

### 🔴 P1 — New findings blocking merge

1. **`Options.Create` bypasses IConfiguration** (`Abstractions/Setup.cs:23`) — options cannot be bound from appsettings.json; breaks IOptionsMonitor; inconsistent with every other Headless.* package.

2. **Filter caching doc/code mismatch** (`AuditLogOptions.cs:38,45`) — `EntityFilter` and `PropertyFilter` docs say "result is cached" but delegates are invoked raw on every SaveChanges. Consumers writing expensive predicates will cause production perf incidents; stateful filters create audit compliance bypass risk.

3. **Transform + null transformer: silent Redact fallback** (`EfAuditChangeCapture.cs:295-336`) — configuring `SensitiveDataStrategy.Transform` without setting `SensitiveValueTransformer` silently produces `"***"` with no error, warning, or startup validation. GDPR compliance requirement silently unmet.

### 🟡 P2 — Should fix before GA

- `required string?` semantic confusion on `AuditLogEntryData` nullable fields
- `JsonElement` round-trip type fidelity lost on OldValues/NewValues read-back
- `_FallbackToRedact` swallows transformer exceptions without logging
- `_DetermineAction` O(n) double-scan via `FirstOrDefault` — fold into main loop
- `IpAddress`/`UserAgent` fields always null in auto-capture path
- `ConfigureAuditLog` double-registration silently last-wins
- **No read abstraction** — `IAuditLogStore` write-only; no `IReadAuditLog` interface
- `DefaultExcludedProperties` hardcoded and not overridable by consumers

### 🔵 P3 — Nice-to-Have

- Extract 8 action string magic literals to named constants
- `_ApplySensitiveValues` redundant `changeType` switch re-derivation
- Defer `List<AuditLogEntryData>` allocation until first auditable entity
- `IsEnabled = false` produces no startup/health signal
- `CorrelationId` no length guard before `HasMaxLength(128)` storage
- Composite key `EntityId` comma-join ambiguous for values containing commas
- `AuditAllEntities` naming reads backwards — consider `AuditByDefault`
