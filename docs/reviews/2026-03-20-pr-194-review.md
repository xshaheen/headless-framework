## Overall Assessment

The core design is **sound**: per-group state machine, callback-based notification,
`System.Threading.Timer` for HalfOpen transitions, lock-per-group for compound transitions,
and OTel instrumentation are all correct architectural choices. The test coverage is thorough
for happy paths.

However, **3 functional correctness bugs** were found that will silently misbehave in production:
1. The `WithCircuitBreaker()` per-consumer override API does nothing at runtime
2. Circuit recovery after open/close cycle silently kills consumers (no restart of listening tasks)
3. EscalationLevel never resets despite configuration and documentation

## Key Findings

### üî¥ P1 ‚Äî Blocks Merge

1. **Per-consumer CB overrides silently ignored** ‚Äî `WithCircuitBreaker(cb => cb.FailureThreshold=3)` is a complete no-op at runtime. `ConsumerCircuitBreakerRegistry` is populated but never read by `CircuitBreakerStateManager`. (`003-pending-p1-...`)

2. **EscalationLevel never resets** ‚Äî `SuccessfulCyclesToResetEscalation` option has no implementation. `EscalationLevel` grows unbounded; at extreme levels causes `OverflowException` in `_TransitionToOpen` under the group lock. (`006-pending-p1-...`)

3. **_ResumeGroupAsync doesn't restart consumer task loops** ‚Äî `_PauseGroupAsync` cancels the CTS, killing all `ListeningAsync` tasks. `_ResumeGroupAsync` creates a new CTS and calls `ResumeAsync` on clients but never restarts the `Task.Factory.StartNew` loops. After the first circuit open/close cycle, all consumers (SQS, NATS, InMemory, Pulsar, Redis) are permanently dead ‚Äî messages silently dropped. (`016-pending-p1-...`)

4. **TOCTOU race on volatile bool + ManualResetEventSlim** ‚Äî All 6 transport `PauseAsync`/`ResumeAsync` implementations (5 MRES + Azure Service Bus) use non-atomic check-then-act. Concurrent calls can leave gate open despite paused state, bypassing the circuit breaker. (`018-pending-p1-...`)

5. **HalfOpenProbeCount is a public no-op** ‚Äî Public `init` property with XML doc explicitly stating the state manager ignores it. Cannot be un-shipped after NuGet publish. (`020-pending-p1-...`)

### üü° P2 ‚Äî Should Fix

6. **Global IsTransientException predicate never called** ‚Äî Hardcoded `CircuitBreakerDefaults.IsTransient` used instead of `_options.IsTransientException`. (`002-pending-p2-...`)
7. **State field not volatile** ‚Äî `IsOpen` lockless read is unsafe on ARM64 without `Volatile.Read`. (`007-pending-p2-...`)
8. **onResume exception swallowed silently** ‚Äî Circuit frozen in HalfOpen with no observable signal if resume callback throws. (`009-pending-p2-...`)
9. **NATS gate.Wait() without CT** ‚Äî Blocks NATS dispatcher thread for full open duration. (`012-pending-p2-...`)
10. **SubscribeExecutor non-nullable CB dependency** ‚Äî Inconsistent with optional pattern in ConsumerRegister and MessageNeedToRetryProcessor. (`014-pending-p2-...`)
11. **ConsumerCircuitBreakerOptions uses set not init** ‚Äî Mutable post-construction, inconsistent with framework style. (`017-pending-p2-...`)
12. **ICircuitBreakerStateManager is internal** ‚Äî No public API to observe circuit state from application code. (`019-pending-p2-...`)
