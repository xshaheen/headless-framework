# Jobs Dashboard vs Messaging Dashboard — Architecture Reference

## High-Level Comparison

| Aspect                 | Jobs Dashboard                          | Messaging Dashboard                       |
|------------------------|-----------------------------------------|-------------------------------------------|
| Package                | Headless.Jobs.Dashboard                 | Headless.Messaging.Dashboard              |
| Frontend               | Vue 3 / TS / Vuetify 3 / Pinia / Vite 6| Vue 3 / TS / Vuetify 3 / Pinia / Vite 6  |
| Backend endpoints      | MapGroup Minimal API (static)           | MapGroup Minimal API (static)             |
| Auth                   | Shared: Headless.Dashboard.Authentication (5 modes) | Same shared package              |
| Realtime               | SignalR                                 | Polling                                   |
| Multi-node model       | Shared DB + Redis heartbeats            | Gateway proxy + Consul/K8s discovery      |
| Node discovery         | Not needed (shared DB)                  | INodeDiscoveryProvider (Consul or K8s)    |
| Tests                  | None yet                                | Unit tests (xUnit + AwesomeAssertions)    |

## Shared Auth Package: Headless.Dashboard.Authentication

Both dashboards reference this package. Zero domain dependencies.

Types:
- AuthMode enum: None, Basic, ApiKey, Host, Custom
- AuthConfig: credentials, API key, custom validator, session timeout
- IAuthService / AuthService: authenticate requests, return AuthResult
- AuthMiddleware: protects /api/* paths, skips static files + /negotiate + /api/auth/*
- AuthResult: success/failure with username
- AuthInfo: mode/enabled/timeout for frontend config

Auth flow:
1. AuthMiddleware intercepts API requests
2. Resolves IAuthService from DI
3. Calls AuthenticateAsync(HttpContext)
4. Returns 401 or sets auth items on context

## Gateway Proxy (Messaging Only)

Why Jobs doesn't need it: single shared DB gives cross-node visibility.
Why Messaging needs it: each node has its own storage; dashboard must proxy to view other nodes.

Components:
- GatewayProxyAgent: routes requests to correct node via cookie (messaging.node)
- GatewayProxyEndpointFilter: IEndpointFilter applied to protected API group
- INodeDiscoveryProvider: Consul (ConsulNodeDiscoveryProvider) or K8s (K8sNodeDiscoveryProvider)
- IRequestMapper + IHttpRequester: map and forward HTTP requests

Node switching: cookie-based (messaging.node + messaging.node.ns for K8s).

## Endpoint Patterns

### Jobs Dashboard Endpoints (MapGroup, /api prefix)
Auth: /api/auth/info, /api/auth/validate, /api/auth/logout
Time Jobs: CRUD, trigger, enable/disable
Cron Jobs: CRUD, trigger, enable/disable
Operations: /api/operations
Host: /api/host/info
Stats: /api/stats
SignalR: /jobs-hub

### Messaging Dashboard Endpoints (MapGroup, /api prefix)
Auth: /api/auth/info, /api/auth/validate (always anonymous)
Metrics: /api/metrics-realtime, /api/metrics-history, /api/stats, /api/meta
Published: /api/published/{status}, /api/published/message/{id}, /api/published/requeue, /api/published/delete
Received: /api/received/{status}, /api/received/message/{id}, /api/received/reexecute, /api/received/delete
Subscribers: /api/subscriber
Nodes: /api/nodes, /api/list-ns, /api/list-svc/{namespace}
Health: /api/health, /api/ping (always anonymous)

## Middleware Pipeline

Both use app.Map(basePath) branching:
1. Static files (EmbeddedFileProvider) — FIRST, before auth
2. Routing
3. CORS
4. AuthMiddleware (only /api/*, skips static + negotiate + auth endpoints)
5. Custom middleware (optional)
6. Endpoints (MapGroup Minimal API)
7. Post-middleware (optional)
8. SPA fallback (404 → index.html with injected config)

Jobs has SignalR hub mapping after endpoints.
Messaging has no SignalR (polling only).

## DI Registration

### Jobs
```
services.AddDashboard(config => { ... })
```
Registers: AuthConfig, IAuthService, DashboardOptionsBuilder, SignalR hub, repositories.

### Messaging
```
options.UseDashboard(d => {
    d.WithBasicAuth("admin", "pass");
    d.SetBasePath("/messaging");
});
```
Registers via IMessagesOptionsExtension:
- MessagingDashboardOptionsBuilder (singleton)
- AuthConfig + IAuthService (from shared package)
- GatewayProxyEndpointFilter
- MessagingMetricsEventListener
- IStartupFilter (auto-injects middleware pipeline)

### K8s Standalone
```
services.AddMessagingDashboardStandalone(
    configure: d => d.WithNoAuth(),
    k8SOption: k => k.ShowOnlyExplicitVisibleNodes = true
);
```

## HTML/Config Injection

Both inject runtime config into index.html <head>:
- Jobs: window.JobsConfig = { basePath, auth, ... }
- Messaging: window.MessagingConfig = { basePath, auth, statsPollingInterval }

Uses <base href="..."/> + inline <script> injection.
