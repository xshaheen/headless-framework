# Headless.Tus.Demo

Complete resumable-upload demo: an ASP.NET Core backend using `tusdotnet` + `Headless.Tus.Azure`
(Azure Block Blob store, registered via `AddTusAzureStore`) and a React frontend driving the tus
1.0.0 protocol with `tus-js-client`.

## What it demonstrates

- **Resumable uploads** — 5 MB PATCH chunks with pause/resume; interrupted uploads (killed tab,
  dropped connection) resume from the last committed byte via tus fingerprints.
- **`AddTusAzureStore` DI registration** — options bound from the `Tus` configuration section with
  FluentValidation at startup; the store is resolved from DI inside the `MapTus` configuration factory.
- **Verbatim `Upload-Metadata` round-trip** — upload a file with a non-ASCII name (e.g. Arabic) and
  watch HEAD echo the header byte-for-byte; downloads carry the name via RFC 8187 `filename*`.
- **Expiration** — unfinished uploads expire after 30 idle minutes (sliding) and a background
  service removes them; completed uploads are never reaped.
- **Termination** — cancel buttons issue tus `DELETE`, discarding server-side data.

## Run it

Prerequisites: Docker (for Azurite), Node 20+ (frontend build).

```bash
# 1. Storage emulator
docker run -d --name azurite -p 10000:10000 mcr.microsoft.com/azure-storage/azurite azurite-blob --blobHost 0.0.0.0

# 2. Frontend (one-time, or after frontend changes) — emits into wwwroot/
cd demo/Headless.Tus.Demo/frontend && npm install && npm run build && cd ..

# 3. Backend (serves the built SPA at http://localhost:5273)
dotnet run
```

Frontend development with hot reload (backend running separately):

```bash
cd demo/Headless.Tus.Demo/frontend
npm run dev   # http://localhost:5173, proxies /files and /api to :5273
```

To run against a real storage account instead of Azurite, set the
`ConnectionStrings:AzureStorage` configuration value.

## Surface

| Route | What |
|---|---|
| `POST/PATCH/HEAD/DELETE /files` | The tus 1.0.0 endpoint (`tusdotnet`, all extensions enabled, 2 GB cap) |
| `GET /api/files` | App-level listing (the tus protocol has no listing operation) — reads the blob container and the store's documented `tus_*` metadata keys |
| `GET /api/files/{id}/download` | Streams a completed upload via `ITusReadableStore` |
| `/` | The built React SPA (when `wwwroot/` exists) |

## Notes

- The Blob service version is pinned (`V2024_11_04`) so the demo works against Azurite, which lags
  behind the newest Azure Storage API versions.
- Single-node demo → tusdotnet's in-process file lock. For multi-node deployments register
  `Headless.Tus.DistributedLocks` and set `FileLockProvider` in the tus configuration.
- `wwwroot/` is generated by Vite and gitignored; the backend shows a hint at `/` when it is missing.
